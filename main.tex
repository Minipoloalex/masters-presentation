\subsection{Reinforcement Learning}
\begin{frame}{Reinforcement Learning (RL)}
  \slidetime{00:15}
  \begin{columns}[T,onlytextwidth]
    \column{0.55\textwidth}
    \centering
    \includegraphics[width=\linewidth]{figures/basic-rl-diagram.pdf}
    \column{0.35\textwidth}
    \begin{itemize}
      \item Effective in complex decision problems
      \item No single algorithm outperforms others in all types of problems
      \item Requires heavy tuning and configuration
    \end{itemize}
  \end{columns}
  \note{
    \begin{itemize}
      \small
      \item \textbf{What is RL}: paradigm to solve \textbf{sequential decision problems} (tomadas de decisão sequencial)
      \item \textbf{Compare with other ML problems}: we have a more complex setup because of the \textbf{sequential} nature of decisions
      \item \textbf{Diagram and MDP-style definitions}: agent and environment, interaction between them. \textbf{On each timestep}, the agent receives information about the \textbf{environment state} and decides what \textbf{action to perform} using what is called its policy. Then, it also \textbf{receives rewards} which help adjust the policy towards the best actions in a state.
      \item \textbf{Different algorithms}: algorithms of diffeerent types that use different ideas. No algorithm is best in all environments.
      \item Solutions to RL problems \textbf{require heavy tuning}: trying many algorithms and parameters
    \end{itemize}
  }
\end{frame}

\subsection{Instance Space Analysis}
\begin{frame}{Instance Space Analysis (ISA)}
  \slidetime{01:15}
  \centering
  \includegraphics[width=0.75\linewidth]{figures/fw-isa.pdf}
  \note{
    \textbf{What is ISA}: Methodology to analyze the performance of algorithms
    \begin{itemize}
      \small
      \item \textbf{Problem and Algorithm Space}
      \item \textbf{Problem Subset}: Instances
      \item \textbf{Feature Space}: Extract features from selected instances
      \item \textbf{Performance Space}: measure performance of algorithms on instances
      \item \textbf{Dataset of features and performance metrics}
      \item \textbf{Select Features \& Dimension Reduction}: Select most relevant features and use a custom ISA (específico) dimension reduction method to reduce to 2D and enable interpretable visualizations.
      \item \textbf{Get Results from ISA}: Algo Selection and performance metrics, instance space visualizations and algorithm footprints.
      \item \textbf{Interpret ISA results}: Iterate on the selected instances, used features, and algorithms.
    \end{itemize}
  }
\end{frame}

\subsection{Metafeatures}
\begin{frame}{Metafeatures}
  \slidetime{02:55}
  High-level measurable characteristics that describe a task. Should be:
  \begin{itemize}
    \item Highly discriminative
    \item Efficiently computable
  \end{itemize}
  \vspace{0.5em}
  
  Classification and regression have well-established metafeatures.

  ISA studies have used these metafeatures to study those problems.
  \note{
    To perform ISA, we \textbf{need features} (like we described before):

    Those are typically called metafeatures:
    \begin{itemize}
      \item \textbf{Definition}: Measurable \textbf{characteristics} that describe a problem and should help \textbf{distinguish algorithm performance}
      \item Should also be \textbf{efficient} since that computational cost limits the gains from algorithm selection
      \begin{itemize}
        \item If these metafeatures take long to compute, then we lose some of the algorithm selection gains. If they take as long as training algorithms, then we could just train the algorithms and select the best one.
      \end{itemize}
    \end{itemize}
    \textbf{Domains} like \textbf{classication and regression} have \textbf{well-established metafeatures} and several studies have used them to apply ISA successfully.
  }
\end{frame}

\section{Literature Review}
\begin{frame}{Literature Review}
  \slidetime{03:30}
  A few issues to apply ISA to RL:
  \begin{itemize}
    \item No established metafeature set
    \item No fixed dataset, only environment interactions
  \end{itemize}
  \vspace{0.3em}
  Many works study properties that impact RL algorithm performance:
  \begin{itemize}
    \item Complexity Measures: capture the difficulty to solve a specific RL problem, %
    but are not efficiently computable for all problems
    \item Benchmarks: isolate environment properties and evaluate their impact on algorithms
  \end{itemize}

  Implicit knowledge of what impacts algorithm performance but no clear computable measures

  \note{
    \begin{itemize}
      \item Opposed to other problems: RL has no established metafeatures
      \item \textbf{no fixed dataset} where we can apply statistical measures
      \item Still, several works study properties that impact RL performance:
      \begin{itemize}
        \item Complexity Measures: study properties that \textbf{make it difficult to solve} RL problems
        \item Benchmarks: vary environment properties that affect their performance and evaluate their impact on performance 
      \end{itemize}
    \end{itemize}
  }
\end{frame}

\subsection{Metafeature Design}

\begin{frame}{Metafeature Design}
  \slidetime{08:45}
  To extract a dataset of trajectories, we use two simple non-learned %
  ``probe'' policies:
  \begin{itemize}
    \item a random actions policy
    \item a rule-based heuristic baseline which depends on the environment
  \end{itemize}
  \vspace{0.5cm}
  Several groups of metafeatures:
  \begin{itemize}
    \item Environment structure
    \item Probe-performance and ego-dynamics
    \item Probe observation-stream
    \item Surrounding-traffic behavior
    \item Environment dynamics
  \end{itemize}
  \note{
    \begin{itemize}
      \item Beyond (para além) training agents, we need to characterize instances to capture their properties.
      \item There is no fixed dataset, so we collect trajectories to make a dataset using 2 simple policies (\textbf{<see slide>}), to \textbf{ensure efficiency}.
    \end{itemize}

    We extract different types of metafeatures, including:
    \begin{enumerate}
      \item observation, action size
      \item reward statistics (mean, standard deviation)
      \item observation statistics
      \item statistics of other vehicles
      \item environment dynamics (PT: da dinamica do ambiente): for example, information about the transitions of the environment
    \end{enumerate}
  }
\end{frame}

\subsection{Performance Normalization}

\begin{frame}{Performance Normalization}
  \slidetime{09:40}
No general performance metric capable of comparing agent performance across heterogeneous environments.

\begin{itemize}
  \item Higher mean return may not indicate an easier problem
  % \item An environment that yields higher mean returns may actually be harder to solve.
\end{itemize}

\vspace{0.45em}
To define performance, we use the random and baseline heuristic policies:
\[
s_{i,a} = \frac{R_{i,a} - R_{i,random}}{R_{i,baseline}-R_{i,random}}
\]
\begin{itemize}
  \item $s_{i,a}$: normalized performance metric for algorithm $a$
  \item $R_{i,a}$, $R_{i,random}$, $R_{i,baseline}$: mean episodic returns for the algorithm $a$, %
the random policy and the baseline heuristic policy, respectively.
\end{itemize}

\note{
  \begin{itemize}
    \item Have performances and metafeatures, but cannot compare performances in \textbf{different environments}.
    \item We cannot use rewards.
    \begin{itemize}
      \item Because the reward scale may be very different.
    \end{itemize}

    \item So, to avoid this, we use this formula to normalize performance.
    \begin{itemize}
      \item uses the \textbf{simple policies} we also used for metafeatures.
      \item Here, the heuristic policy functions as a baseline.
    \end{itemize}
    \item (<Explain meaning of example values --- close to 0 and 1>)
    \begin{itemize}
      \item For example, a value close to 1 means \textbf{good agent performance}, similar to the heuristic policy.
      \item A low value closer to 0 means the \textbf{environment was harder} for the agent.
    \end{itemize}
  \end{itemize}
}
\end{frame}

\subsection{Results Discussion}
\begin{frame}{Results Discussion}
  \begin{itemize}
    \item Single-environment projections may lack instance diversity
    \item Analyzing individual environments provides context for multi-environment projections
    \item Results based on multiple environments yield better instance space coverage and provide more generalizable results
    \item The algorithm selection methodology gains significant performance over the best-performing algorithms for some projections
    \item A few projections yield weaker selection performance, likely due to the predominance of a single algorithm, or because the metafeatures are not discriminative enough
  \end{itemize}
  \note{
    To conclude the results:
    \begin{itemize}
      \item First we saw an \textbf{example} of a projection with \textbf{only one environment}
      \begin{itemize}
        \item these projections \textbf{have less diversity} but \textbf{can give more detailed context}
      \end{itemize}
      \item In contrast, \textbf{multi-environment projections} provide \textbf{more generalizable results}
      \item In some cases, selection has good performance gains relative to the best algorithm.
      \item In some projections, performance was \textbf{more limited}:
      \begin{itemize}
        \item Single algorithm dominance
        \item Metafeatures not able to distinguish algorithms sufficiently for the selection to be able to improve performance
      \end{itemize}
    \end{itemize}
  }
